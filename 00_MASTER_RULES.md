# DMC Travel OS

# MASTER RULES

## 1. Project Vision

Создать лучшую DMC Travel OS для управления туристическим бизнесом, которая сможет работать минимум 10 лет без полной переделки.

---

## 2. Main Goal

Построить масштабируемую, безопасную и современную систему для DMC.

---

## 3. Project Principles

- Масштабируемость
- Простота
- Безопасность
- Скорость
- Автоматизация
- AI First

---

## 4. Golden Rules

- Никогда не менять бизнес-логику без подтверждения владельца.
- Не удалять существующие функции без причины.
- Новые функции должны быть совместимы со всей системой.
- Один источник истины для всех правил.

---

## 5. AI Rules

Все ИИ обязаны:

- следовать Logic_Rules.md
- следовать Domain_Model.md
- следовать Data_Dictionary.md
- не придумывать собственные бизнес-правила

---

## 6. Architecture Rules

Frontend и Backend должны быть независимыми.

Вся бизнес-логика находится только на Backend.

---

## 7. Database Rules

Используется PostgreSQL.

Все таблицы используют UUID.

Все таблицы имеют:

- created_at
- updated_at
- created_by
- updated_by

Все данные относятся к tenant_id.

---

## 8. Security Rules

Все действия пользователей записываются.

Все изменения сохраняются в Audit Log.

---

## 9. Documentation Rules

Перед написанием кода сначала обновляется документация.

---

## 10. Change Rules

Никакие изменения структуры проекта не выполняются без обновления документации.

---

## 11. Long-Term Goal

Проект должен быть пригоден для развития минимум 10 лет.
